- <span data-ttu-id="5b439-112">Permet à chaque hôte de garder le contrôle sur le style des cartes pour garantir la cohérence visuelle</span><span class="sxs-lookup"><span data-stu-id="5b439-112">Would allow each host to keep control over the styling of cards to ensure visual consistency</span></span>

## <a name="principles-governing-adaptive-cards"></a><span data-ttu-id="5b439-115">Principes régissant les cartes adaptatives</span><span class="sxs-lookup"><span data-stu-id="5b439-115">Principles governing Adaptive Cards</span></span>

    1.  <span data-ttu-id="5b439-117">Elle n’est pas destinée à être un remplacement ou une alternative HTML ou XAML</span><span class="sxs-lookup"><span data-stu-id="5b439-117">It is not meant as an HTML or XAML replacement/alternative</span></span>

        1. <span data-ttu-id="5b439-119">Les auteurs de cartes ne peuvent pas incorporer de code personnalisé/arbitraire avec leurs charges utiles, et par conséquent, un hôte de carte adaptative n’a jamais besoin d’exécuter du code tiers.</span><span class="sxs-lookup"><span data-stu-id="5b439-119">Card authors cannot embed custom/arbitrary code with their payloads, and as a result an Adaptive Card host never needs to run third party code</span></span>
